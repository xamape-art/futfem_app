FUTFEM_APP

App independiente para recoger y visualizar estadísticas de jugadoras de las ligas femeninas de la FCF — extensible a múltiples divisiones.

Stack: React 19 + Vite + Supabase
Deploy: GitHub → Vercel
Sync automático: GitHub Actions

🎯
Resumen ejecutivo §1
ℹ️
FUTFEM_APP replica la infraestructura de la sección Datos de ATClub (scraping FCF → Supabase → React) pero como app totalmente independiente, sin lógica de equipo propio y preparada para múltiples ligas/divisiones.
🚀

Qué hace la app

Muestra estadísticas acumuladas de jugadoras (partidos, minutos, goles, tarjetas) extraídas automáticamente de las actas oficiales de la FCF. Sin login, 100% pública.

🔗

Independencia total

Repo GitHub propio · Proyecto Supabase propio · Deploy Vercel propio. No comparte nada con ATClub — ni BD ni secrets ni dominio.

🏟️

Multi-liga desde el inicio

Tabla leagues en Supabase como maestro. Añadir una nueva liga = insertar una fila + una línea en el workflow. Cero cambios de código.

⚙️

Sync automático

GitHub Actions ejecuta el script de scraping cada lunes 10:00 UTC para todas las ligas activas. También disparo manual desde Actions UI.

4.3 Ejecutar schema SQL — Dashboard → SQL Editor → New query

Copiar y ejecutar el siguiente SQL completo de una vez: 

-- ═══════════════════════════════════════════════════════════════
-- FUTFEM_APP · Schema v1
-- Ejecutar en: Supabase Dashboard → SQL Editor
-- ═══════════════════════════════════════════════════════════════

-- ── 1. leagues — master de ligas/divisiones FCF ─────────────────
CREATE TABLE IF NOT EXISTS leagues (
  id          UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  name        TEXT        NOT NULL,          -- 'Tercera Federació - Grup V'
  short_name  TEXT        NOT NULL,          -- '3ª Fed Gr.V'
  group_path  TEXT        NOT NULL UNIQUE,   -- segmento de URL FCF
  fcf_seasons TEXT[]      NOT NULL DEFAULT '{}', -- ['2526','2627']
  active      BOOLEAN     NOT NULL DEFAULT true,
  sort_order  INT         NOT NULL DEFAULT 0,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- ── 2. fcf_stats — stats acumuladas por jugadora ─────────────────
CREATE TABLE IF NOT EXISTS fcf_stats (
  id              UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  league_id       UUID        NOT NULL REFERENCES leagues(id) ON DELETE CASCADE,
  season          TEXT        NOT NULL,    -- '25-26' | '26-27'
  team_slug       TEXT        NOT NULL,
  team_name       TEXT        NOT NULL,
  player_fcf_name TEXT        NOT NULL,    -- 'RIBAS CAELLES, LAURA'
  dorsal          INT,
  partidos        INT         NOT NULL DEFAULT 0,
  titular         INT         NOT NULL DEFAULT 0,
  suplente        INT         NOT NULL DEFAULT 0,
  minutos         INT         NOT NULL DEFAULT 0,
  goles           INT         NOT NULL DEFAULT 0,
  amarillas       INT         NOT NULL DEFAULT 0,
  rojas           INT         NOT NULL DEFAULT 0,
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  CONSTRAINT fcf_stats_unique UNIQUE (league_id, season, team_slug, player_fcf_name)
);

CREATE INDEX IF NOT EXISTS idx_fcf_stats_league_season ON fcf_stats(league_id, season);
CREATE INDEX IF NOT EXISTS idx_fcf_stats_team ON fcf_stats(league_id, season, team_slug);

-- ── 3. actas_procesadas — idempotencia del sync ──────────────────
CREATE TABLE IF NOT EXISTS actas_procesadas (
  id            UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  league_id     UUID        NOT NULL REFERENCES leagues(id) ON DELETE CASCADE,
  season        TEXT        NOT NULL,
  fcf_season    TEXT        NOT NULL,   -- '2526' | '2627'
  acta_url      TEXT        NOT NULL UNIQUE,
  jornada       INT,
  local_slug    TEXT,  visitant_slug   TEXT,
  local_name    TEXT,  visitant_name   TEXT,
  players_count INT         DEFAULT 0,
  processed_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_actas_league_season ON actas_procesadas(league_id, season);

-- ── 4. RLS — SELECT público, writes solo via service_role ────────
ALTER TABLE leagues          ENABLE ROW LEVEL SECURITY;
ALTER TABLE fcf_stats        ENABLE ROW LEVEL SECURITY;
ALTER TABLE actas_procesadas ENABLE ROW LEVEL SECURITY;

CREATE POLICY "leagues_read"
  ON leagues FOR SELECT USING (true);
CREATE POLICY "fcf_stats_read"
  ON fcf_stats FOR SELECT USING (true);
CREATE POLICY "actas_procesadas_read"
  ON actas_procesadas FOR SELECT USING (true);

-- ── 5. Seed inicial de ligas ─────────────────────────────────────
INSERT INTO leagues (name, short_name, group_path, fcf_seasons, active, sort_order)
VALUES (
  'Tercera Federació - Grup V',
  '3ª Fed Gr.V',
  'futbol-femeni/tercera-federacio-futbol-femeni/grup-v',
  ARRAY['2526', '2627'],
  true, 1
)
ON CONFLICT (group_path) DO NOTHING;

Lógica de resolución de liga (nueva función)
// Al inicio del script, resuelve el league_id desde Supabase
async function resolveLeague(groupPath) {
  const { data, error } = await supabase
    .from('leagues')
    .select('id, name, group_path')
    .eq('group_path', groupPath)
    .single();

  if (error || !data) {
    console.error(`Liga no encontrada: ${groupPath}`);
    process.exit(1);
  }
  return data; // { id: UUID, name: string, group_path: string }
}

App.tsx

Sin auth. Carga ligas al mount, luego allStats+actas en paralelo al cambiar liga/temporada. Todo el filtrado es en memoria.

Flujo de datos en App.tsx
// 1. Mount: cargar ligas
supabase.from('leagues').select('*').eq('active', true).order('sort_order')
→ setLeagues(data) → auto-select leagues[0]

// 2. Al cambiar liga: derivar temporadas del objeto league
league.fcf_seasons.map(fcfSeasonToApp)  // '2627' → '26-27'
→ setAvailableSeasons() → auto-select seasons[0]

// 3. Al cambiar (leagueId, season): cargar datos en paralelo
await Promise.all([
  supabase.from('fcf_stats')
    .select('*').eq('league_id', leagueId).eq('season', season),
  supabase.from('actas_procesadas')
    .select('*').eq('league_id', leagueId).eq('season', season)
])
→ setAllStats(stats) → setActas(actas)
→ setTeams(dedup por team_slug)   // sin llamada extra a BD

// 4. Al cambiar team: filtrar allStats en memoria
setTeamStats(allStats.filter(s => s.team_slug === selectedTeam))

// 5. Búsqueda global (useMemo)
searchResults = allStats.filter(s =>
  s.player_fcf_name.toLowerCase().includes(query.toLowerCase())
)

GitHub Actions workflow §9
.github/workflows/sync-actas.yml
name: Sync Actas FCF · FUTFEM

on:
  # Automático: cada lunes 10:00 UTC
  schedule:
    - cron: '0 10 * * 1'

  # Manual: desde GitHub Actions UI
  workflow_dispatch:
    inputs:
      league:
        description: 'group_path FCF (obligatorio)'
        required: true
        default: 'futbol-femeni/tercera-federacio-futbol-femeni/grup-v'
        type: string
      season:
        description: 'Código temporada FCF'
        required: true
        default: '2627'
        type: choice
        options: ['2627', '2526']
      force:
        description: 'Reimportar actas ya procesadas'
        default: 'false'
        type: boolean
      dry_run:
        description: 'Solo parsear, no escribir en Supabase'
        default: 'false'
        type: boolean

jobs:
  # ─── PROGRAMADO: todas las ligas activas ────────────────────────
  sync-scheduled:
    if: github.event_name == 'schedule'
    name: Sync ${{ matrix.league }}
    runs-on: ubuntu-latest
    timeout-minutes: 30
    strategy:
      fail-fast: false
      matrix:
        # ← Añadir nuevas ligas aquí (group_path)
        league:
          - 'futbol-femeni/tercera-federacio-futbol-femeni/grup-v'
        season:
          - '2627'
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '24', cache: 'npm' }
      - run: npm ci
      - name: Run sync
        run: |
          node scripts/sync-actas.js \
            --league "${{ matrix.league }}" \
            --season "${{ matrix.season }}"
        env:
          SUPABASE_URL:              ${{ secrets.SUPABASE_URL }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}

  # ─── MANUAL: una liga + temporada específica ────────────────────
  sync-manual:
    if: github.event_name == 'workflow_dispatch'
    name: Manual Sync
    runs-on: ubuntu-latest
    timeout-minutes: 30
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '24', cache: 'npm' }
      - run: npm ci
      - name: Run sync
        run: |
          FLAGS="--league ${{ inputs.league }} --season ${{ inputs.season }}"
          [ "${{ inputs.force }}"   = "true" ] && FLAGS="$FLAGS --force"
          [ "${{ inputs.dry_run }}" = "true" ] && FLAGS="$FLAGS --dry-run"
          node scripts/sync-actas.js $FLAGS
        env:
          SUPABASE_URL:              ${{ secrets.SUPABASE_URL }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}
💡
Para añadir una nueva liga al cron automático: añadir su group_path bajo matrix.league en el workflow. No hace falta tocar ningún otro fichero de código.
